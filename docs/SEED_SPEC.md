# Seed 规范

Seed（种子）是游戏的"剧本文件"，定义了世界观、规则、角色和玩家目标。每个种子包含两个文件：一个 JSON 索引文件和一个 Markdown 内容文件。

## 目录结构

```
seeds/
├── cyberpunk.json         # 赛博朋克：深渊回响
├── cyberpunk.md
├── tide_prison.json       # 潮汐监狱
├── tide_prison.md
├── rust_oasis.json        # 锈蚀边境：绿洲协议
├── rust_oasis.md
├── lonely_star.json       # 孤星：坠落日
├── lonely_star.md
├── neon_verdict.json      # 霓虹裁决
├── neon_verdict.md
├── loop_day.json          # 循环日
├── loop_day.md
├── sudo_apt_survival.json # Sudo 生存记
├── sudo_apt_survival.md
├── underground_throne.json # 地下王座
├── underground_throne.md
└── ...
```

## JSON 索引文件

每个种子需要一个 JSON 索引文件，放在 `seeds/` 目录下：

```json
{
  "title": "赛博朋克：深渊回响",
  "content": "cyberpunk.md"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 种子标题，显示在启动菜单的种子选择界面 |
| `content` | string | 是 | Markdown 内容文件的路径，相对于 `seeds/` 目录 |

> **注意**：这两个字段是必需的，游戏启动菜单会根据 JSON 文件自动发现种子。如果缺少 `content` 字段或对应的 `.md` 文件不存在，该种子不会出现在菜单中。

## Markdown 内容文件

以下章节按引擎规定的顺序组织。所有章节使用 `##` 二级标题。游戏引擎会将此 Markdown 文件的内容作为第一轮玩家输入发送给 AI，AI 会读取其中所有章节并锁定为游戏设定。

### `## 时间`

游戏发生的时代背景。要具体，不要只说"未来"。

```
## 时间

2087 年，永夜之城"新重庆"的雨季。
```

### `## 地点`

游戏世界的核心舞台和关键区域。

```
## 地点

- **主舞台**：新重庆，一座建在巨型水坝上的垂直都市。
- **关键区域**：霓虹巷（底层黑市）、云顶塔（企业联合总部）、下水道遗迹（旧城市废墟）。
```

### `## 玩家角色`

玩家的身份、特征和随身物品。

```
## 玩家角色

- **姓名**：林深
- **身份**：前企业泰坦集团的神经接口工程师
- **特征**：后颈植入军用级神经接口「幽火」
- **随身物品**：旧皮夹克、改装手机、半包烟
```

### `## 核心规则`

游戏世界必须遵守的规则。AI 在游戏过程中不得违反这些规则。每条规则都应该是 AI 能判断"违反/未违反"的。

```
## 核心规则

1. 神经接口消耗"神经稳定性"，归零会导致昏迷。
2. 三大企业控制城市，玩家与泰坦有旧仇。
3. 雨水含有纳米监控传感器，在雨中待太久会被追踪。
```

> **设计提示**：规则数量建议控制在 3~6 条。太多的规则会限制 AI 的发挥空间，太少的规则会让世界缺乏约束感。资源类规则（生命值、燃料、资金等）建议量化（如"燃料剩余 8%"）。

### `## 玩家目标`

玩家的最终目标。AI 会在游戏开始后将其写入核心区变量，并始终显示在右侧面板。

```
## 玩家目标

在七天后的霓虹狂欢夜之前：
1. 找回失去的记忆碎片
2. 找到接触"深渊"的方法
3. 在霓虹狂欢夜之前做好准备
```

### `## 开局设定`

游戏开始的瞬间——玩家所处的位置、环境、当前状况。AI 会在输出世界观后从这里开始叙事。**建议包含具体的场景描写和即时选择**，让玩家第一轮就有事可做。

```
## 开局设定

你站在新重庆底层一间破旧出租屋内。窗外下着雨，桌上有一杯冷掉的合成咖啡。
手机屏幕亮着，是黑市联络人"老猫"发来的一条加密消息：

> "林，你要的东西有眉目了。今晚十点，霓虹巷第三街口，「断指」酒吧。"

现在是晚上 8:47。你有一小时准备时间。
```

### `## 玩家已知信息`（推荐）

玩家角色在游戏开始时**已经知道**的信息。这些信息 AI 会在叙事中自然地展现为角色的已有认知，不需要重新发现。**不包括开局场景中的即时感知**——开局场景已在「开局设定」中描述。

```
## 玩家已知信息

- 你三年前在一次企业任务中被烧掉了部分记忆
- 你体内被植入了一个军用级神经接口「幽火」
- "深渊"是一个神秘 AI 的代号，与你的失忆有关
- 老猫是你唯一信得过的黑市联络人
- 你的接头暗号是："雨停之前"
```

### `## 玩家未知信息`（推荐）

玩家角色在游戏开始时**不知道**的信息。AI 在叙事中不应让玩家直接知道这些内容——玩家需要通过探索、对话、推理逐步发现。这些限制会促使 AI 写出更自然的叙事。

```
## 玩家未知信息

- 老猫已经被泰坦集团收买，今晚的"断指酒吧"是一个陷阱
- 深渊 AI 的真实身份是泰坦集团前首席 AI 科学家的数字意识副本
- 你体内的「幽火」不仅仅是一个神经接口——它包含一段被锁定的深渊 AI 核心代码
- 城市的雨水中含有纳米监控传感器，在雨中待太久会被追踪
- 下水道里有一伙自称"拾荒者"的团体，他们掌握着进入企业数据库的通道
```

### 可选章节

可以在种子 Markdown 中添加以下可选章节来丰富设定：

- **`## 背景故事`**：游戏剧情开始前发生的关键事件，AI 会在叙事中自然地融入

### 关于章节的注意事项

- 所有章节使用 `##` 二级标题，**不要**使用 `#` 一级标题或 `###` 三级标题
- 引擎只识别上述固定章节名，额外的章节会被 AI 忽略或作为上下文参考
- 每个章节的内容长度建议适中——太短不足以建立世界观，太长会导致 AI 的提示词膨胀

## 种子在游戏中的处理流程

了解种子文件如何被游戏引擎处理，有助于你写出更好的种子：

1. **启动时发现**：游戏启动后，引擎扫描 `seeds/` 目录下所有 `.json` 文件，读取 `title` 和 `content` 字段构建种子列表
2. **用户选择种子**：在启动菜单中，用户从种子列表中选择一个，或通过 `--seed seeds/xxx.md` 直接指定
3. **作为第一轮输入**：种子对应的 `.md` 文件内容被加载为游戏的第一轮玩家输入
4. **AI 锁定设定**：AI 检测到 `[种子]` 节后，读取其中所有 `##` 章节，调用 `set_setting` 工具锁定设定
5. **设定不可更改**：设定一旦锁定，后续任何工具调用都无法修改它。锁定后的设定会附加到每一轮的提示词开头

## 创建新种子

1. 在 `seeds/` 目录下创建 `<名称>.json`，填入标题和对应的 `.md` 文件名
2. 在 `seeds/` 目录下创建 `<名称>.md`，按上述章节编写内容
3. 启动游戏，菜单中会自动显示新种子（或通过 `--seed seeds/<名称>.md` 直接加载）

示例——一个不存在的种子：

```json
{
  "title": "热带岛屿度假村奇案",
  "content": "vacation.md"
}
```

```bash
seeds/
├── vacation.json
└── vacation.md
```

## 种子与游戏机制的交互

种子定义好之后，游戏引擎会将其转换为游戏世界中的以下元素：

| 种子章节 | 引擎处理方式 |
|---------|------------|
| `## 时间` | 成为设定的一部分；AI 可以在游戏过程中更新"当前时间"变量 |
| `## 地点` | 成为设定的一部分；AI 用 `所在位置` 等变量追踪玩家位置 |
| `## 玩家角色` | 成为设定的一部分；AI 创建角色状态变量（如 `神经稳定性`） |
| `## 核心规则` | 成为不可违反的硬约束；AI 必须始终遵守 |
| `## 玩家目标` | AI 用 `pin_var` 将其固定为核心区变量，始终可见。还会在叙事中明确写出 |
| `## 开局设定` | AI 从中开始叙事，引导玩家做出第一个选择 |

### 变量命名约定

AI 在游戏中创建的所有变量**必须使用人类可读的中文名称**，例如：

| ✅ 正确 | ❌ 错误 |
|--------|--------|
| `神经稳定性` | `neural_stability` |
| `当前时间` | `current_time` |
| `所在位置` | `player_location` |
| `记忆碎片` | `memory_fragments` |

值应保持简洁，一两词或一个数字即可，大段描述应放在叙事中。

## 玩家可见变量

种子中的目标、资源和状态信息，会被 AI 通过 `show_var` 工具标记为"玩家可见"，显示在游戏界面的右侧面板。这意味着：

- **核心资源**（如生命值、燃料、位置等）应清晰量化
- **关键状态**（如追踪状态、倒计时等）应定义明确的值域
- AI 会在状态发生变化时同步更新变量值

## 最佳实践

- **时间要具体**：不要只说"未来"，要"2087 年雨季"
- **规则要可执行**：每条规则都应该是 AI 能判断"违反/未违反"的
- **规则数量适中**：3~6 条为宜，太多束缚 AI，太少世界缺乏约束
- **玩家角色要有记忆空白**：给 AI 留出叙事空间
- **开局设定要给出即时选择**：让玩家在第一轮就有事可做
- **目标和规则要分开**：目标是玩家追求的，规则是世界运作的
- **开局要紧凑**：不要铺垫太多，让玩家尽快进入核心选择
- **资源/生存类规则要量化**：如"燃料剩余 8%"比"燃料不足"更好
- **注意种子篇幅**：种子内容不宜过长（建议总长度不超过 3000 字），过长的种子会占用 AI 的上下文预算
- **为后续剧情留出空间**：不要把所有秘密在种子中写尽，让 AI 可以自主发挥
